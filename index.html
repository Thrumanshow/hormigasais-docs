Protocolo LBH

Lenguaje Binario HormigasAIS — Documentación oficial

El Protocolo LBH (Lenguaje Binario HormigasAIS) es el primer sistema de certificación criptográfica de activos digitales desarrollado desde el borde (Edge Computing), operado desde Android/Termux en San Miguel, El Salvador. Sin servidores centralizados. Sin nube. Soberanía total.

¿Qué puedes hacer con LBH?

🔏
Sellar activos
Certifica imágenes, documentos y archivos digitales con una firma CLHQ única e irrepetible.
🔍
Verificar autenticidad
Cualquier persona puede verificar un sello LBH públicamente sin necesidad de cuenta.
🏷️
Badge descargable
Genera un badge PNG con tu firma para pegar en tu sitio web y redes sociales.
🔗
Widget embebible
Integra verificación LBH en vivo en cualquier sitio web con una línea de HTML.

Quickstart

Sella tu primer activo en menos de 2 minutos

1. Verificar que el nodo está activo

# Terminal / cURL curl https://api.hormigasais.com/verify # Respuesta esperada: { "status": "VALIDADO", "signature": "CLHQ-MASTER-KEY", "protocol": "Lenguaje-Binario-HormigasAIS" }

2. Sellar un activo

# POST /seal curl -X POST https://api.hormigasais.com/seal \ -H "Content-Type: application/json" \ -d '{ "owner": "Tu-Nombre", "asset": "mi-logo.png", "hash": "sha256-del-archivo" }' # Respuesta: { "status": "SELLADO", "sello": { "signature": "CLHQ-A8F3K2M1", "owner": "Tu-Nombre", "protocol": "Lenguaje-Binario-HormigasAIS", "timestamp": "2026-05-08T..." } }

3. Verificar el sello

curl https://api.hormigasais.com/seal/CLHQ-A8F3K2M1 # Respuesta: { "status": "VERIFICADO", "sello": { ... } }

4. JavaScript / Fetch

// Sellar un activo desde el navegador const res = await fetch('https://api.hormigasais.com/seal', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ owner: 'Mi-Empresa', asset: 'logo-v2.png', hash: 'abc123...' }) }); const data = await res.json(); console.log(data.sello.signature); // CLHQ-XXXXXXXX

Endpoints

Referencia completa de la API LBH v2.0

GET/verify

Verifica que el nodo soberano A16 está activo y retorna la firma maestra del protocolo. No requiere autenticación.

{ "status": "VALIDADO", "signature": "CLHQ-MASTER-KEY", "origin": "Nodo-Soberano-A16" }
POST/seal

Registra un sello criptográfico para un activo digital. Retorna una firma CLHQ única verificable públicamente.

// Request body { "owner": "string" // Nombre del propietario "asset": "string" // Descripción del activo "hash": "string" // SHA-256 del archivo (opcional) "plan": "free|premium|enterprise" } // Response 200 { "status": "SELLADO", "sello": { "signature": "CLHQ-XXXXXXXX", "owner": "string", "asset": "string", "protocol": "Lenguaje-Binario-HormigasAIS", "nodo": "A16-SanMiguel-SV", "timestamp": "ISO 8601" } }
GET/seal/{firma}

Verifica la autenticidad de un sello por su firma CLHQ. Público y sin autenticación.

// GET /seal/CLHQ-A8F3K2M1 { "status": "VERIFICADO", "sello": { ... } } // Si no existe: { "error": "Sello no encontrado" } // 404
POST/push/{nodo}

Envía una feromona de aprendizaje desde un nodo de la colonia al registro distribuido.

{ "type": "learning", "value": 0.85, "trust": 0.9 }
GET/consensus

Retorna el consenso distribuido de la colonia basado en el promedio de valores de todos los nodos.

{ "consensus": "ACEPTAR", "promedio": "0.7823", "nodos": 9 }

Autenticación

Acceso a la API LBH

Los endpoints públicos (/verify, /seal/{firma}, /all, /consensus) no requieren autenticación.

El endpoint /seal (POST) es público en plan Free. Para planes Premium y Enterprise, próximamente se implementará autenticación via API Key con header:

Authorization: Bearer LBH-APIKEY-XXXXXXXX
Nota: Durante la fase de lanzamiento, todos los endpoints son accesibles sin autenticación para facilitar la adopción del protocolo.

Códigos de respuesta de la API LBH
